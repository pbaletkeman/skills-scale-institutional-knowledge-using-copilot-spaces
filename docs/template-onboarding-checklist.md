# Team Member Onboarding Checklist

## Purpose
This checklist ensures new team members are effectively onboarded to OctoAcme projects with all the context, access, and support they need to contribute successfully.

## Pre-Arrival (Manager/Team Lead)

### Administrative Setup
- [ ] Provision company accounts (email, Slack, etc.)
- [ ] Request hardware and equipment
- [ ] Schedule first day logistics
- [ ] Assign onboarding buddy/mentor
- [ ] Prepare welcome message for the team

### Access and Tools
- [ ] Provision GitHub organization access
- [ ] Add to relevant repositories with appropriate permissions
- [ ] Grant access to project management tools (GitHub Projects, Jira, etc.)
- [ ] Provide access to documentation platforms
- [ ] Set up development environment access
- [ ] Grant access to communication channels (Slack, Teams)
- [ ] Provision design tools (Figma, Sketch) if applicable
- [ ] Set up monitoring and logging tool access

## First Day

### Welcome and Orientation
- [ ] Welcome meeting with manager
- [ ] Introduction to team members
- [ ] Introduction to onboarding buddy
- [ ] Office/remote work tour and logistics
- [ ] Review org chart and team structure

### Initial Setup
- [ ] Set up workstation and hardware
- [ ] Verify access to all systems and tools
- [ ] Join team communication channels
- [ ] Add to team calendar and distribution lists
- [ ] Set up development environment (if applicable)

### Initial Documentation Review
- [ ] Review company values and culture docs
- [ ] Read team charter and working agreements
- [ ] Review OctoAcme project management overview
- [ ] Read relevant role persona documentation
- [ ] Understand communication cadence and meeting rhythms

## First Week

### Project and Process Familiarization
- [ ] Review current projects and roadmap
- [ ] Understand team's project management process
- [ ] Review project documentation in docs/ folder
- [ ] Attend all regular team meetings
- [ ] Shadow team members in their daily work
- [ ] Review recent retrospective notes and action items

### Role-Specific Onboarding
- [ ] Review role-specific responsibilities from octoacme-roles-and-personas.md
- [ ] Understand key interactions with other roles
- [ ] Review role-specific tools and workflows
- [ ] Identify key stakeholders for the role
- [ ] Schedule 1:1s with cross-functional partners

### First Tasks
- [ ] Complete initial training modules
- [ ] Set up local development environment (developers)
- [ ] Complete small starter task or bug fix
- [ ] Participate in code review or design review
- [ ] Ask questions and document learnings

## First Month

### Deepening Knowledge
- [ ] Complete role-specific training
- [ ] Review codebase/product architecture (if applicable)
- [ ] Understand product features and user flows
- [ ] Review technical documentation and ADRs
- [ ] Learn about key integrations and dependencies

### Building Relationships
- [ ] Schedule 1:1s with all team members
- [ ] Meet with key stakeholders outside the team
- [ ] Understand escalation paths and support channels
- [ ] Join relevant communities of practice
- [ ] Attend cross-team meetings as appropriate

### Contributing
- [ ] Take on independent tasks with guidance
- [ ] Contribute to team ceremonies (standups, planning, retros)
- [ ] Participate actively in reviews and discussions
- [ ] Share early feedback and questions
- [ ] Start building domain expertise

### Feedback and Adjustment
- [ ] Weekly check-ins with manager
- [ ] Regular check-ins with onboarding buddy
- [ ] Share onboarding feedback and suggestions
- [ ] Identify knowledge gaps and address them
- [ ] Adjust onboarding plan based on progress

## First Quarter (90 Days)

### Full Team Integration
- [ ] Take ownership of features or areas
- [ ] Lead or co-facilitate team ceremonies
- [ ] Mentor or support newer team members
- [ ] Contribute to process improvements
- [ ] Participate in planning and estimation

### Milestone Achievements
- [ ] Complete major deliverable or feature
- [ ] Demonstrate independent problem-solving
- [ ] Establish regular cadence with stakeholders
- [ ] Understand team's success metrics and goals
- [ ] Align personal goals with team objectives

### 90-Day Review
- [ ] Complete formal onboarding review with manager
- [ ] Discuss progress, strengths, and development areas
- [ ] Set goals for the next quarter
- [ ] Provide comprehensive onboarding feedback
- [ ] Update onboarding materials based on experience

## Role-Specific Onboarding Tasks

### Developers
- [ ] Set up local development environment
- [ ] Complete first code commit and PR
- [ ] Review coding standards and conventions
- [ ] Understand testing strategy and coverage
- [ ] Learn deployment and CI/CD processes
- [ ] Review architecture and design patterns
- [ ] Understand debugging and troubleshooting tools

### Product Managers
- [ ] Review product vision and strategy
- [ ] Understand customer segments and personas
- [ ] Review product roadmap and priorities
- [ ] Learn product analytics and metrics
- [ ] Meet with key customers or stakeholders
- [ ] Review competitive landscape
- [ ] Understand go-to-market strategy

### Project Managers
- [ ] Review active projects and timelines
- [ ] Understand reporting and communication cadence
- [ ] Learn project tracking tools and processes
- [ ] Review risk management approach
- [ ] Understand budget and resource allocation
- [ ] Meet with project sponsors and stakeholders
- [ ] Review historical project documentation

### UX/UI Designers
- [ ] Review design system and component library
- [ ] Understand design tools and workflows
- [ ] Review user research insights and personas
- [ ] Learn accessibility and responsive design standards
- [ ] Understand design review and approval process
- [ ] Review existing product designs
- [ ] Meet with user research team

### Quality Assurance Leads
- [ ] Review test strategy and approach
- [ ] Set up test environment access
- [ ] Understand automated testing framework
- [ ] Review test case management tools
- [ ] Learn defect tracking and triage process
- [ ] Review quality metrics and goals
- [ ] Understand release criteria and quality gates

### DevOps Engineers
- [ ] Review infrastructure architecture
- [ ] Understand CI/CD pipeline configuration
- [ ] Learn monitoring and alerting systems
- [ ] Review incident response procedures
- [ ] Understand infrastructure as code practices
- [ ] Review security and compliance requirements
- [ ] Learn deployment processes and runbooks

### Technical Writers
- [ ] Review documentation structure and organization
- [ ] Understand documentation tools and workflows
- [ ] Review style guide and templates
- [ ] Learn content review and approval process
- [ ] Understand documentation release cycle
- [ ] Review user feedback and support tickets
- [ ] Meet with SMEs and stakeholders

### Support/Customer Success
- [ ] Review customer support processes
- [ ] Learn ticketing and CRM systems
- [ ] Understand escalation procedures
- [ ] Review knowledge base and FAQs
- [ ] Shadow experienced support team members
- [ ] Learn product features and common issues
- [ ] Review customer communication guidelines

## Onboarding Success Metrics

- Time to first contribution
- Time to independent productivity
- Onboarding satisfaction score
- Knowledge retention assessment
- 30-60-90 day milestone achievement
- Manager and peer feedback

## Tips for Success

### For New Team Members
- Ask questions early and often
- Take notes and document your learnings
- Be proactive in scheduling meetings
- Share feedback on the onboarding process
- Build relationships across the team
- Set realistic expectations for yourself

### For Managers and Buddies
- Schedule regular check-ins
- Provide clear, actionable feedback
- Create safe space for questions
- Assign appropriate starter tasks
- Celebrate early wins
- Be patient and supportive

### For Teams
- Welcome new members warmly
- Make time for introductions and questions
- Include new members in team activities
- Share context and history when relevant
- Provide constructive feedback
- Foster psychological safety
