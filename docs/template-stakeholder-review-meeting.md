## Review Types and Variations

### Sprint/Iteration Review
**Focus:** Demo completed work, gather feedback
**Frequency:** Every sprint (1-2 weeks)
**Duration:** 30-60 minutes
**Attendees:** Core team + immediate stakeholders

### Phase Gate Review
**Focus:** Major milestone completion, go/no-go decision
**Frequency:** At phase completion (Project Initiation, Planning, Execution, Release)
**Duration:** 60-90 minutes
**Attendees:** All stakeholders + sponsors

### Monthly Stakeholder Update
**Focus:** High-level progress, key metrics, major decisions
**Frequency:** Monthly
**Duration:** 30-45 minutes
**Attendees:** Executive stakeholders, sponsors

### Design Review
**Focus:** Review designs, gather feedback on user experience
**Frequency:** As needed during design phase
**Duration:** 60 minutes
**Attendees:** Designers, product, key stakeholders, users if possible

### Technical Review
**Focus:** Architecture decisions, technical approach
**Frequency:** As needed for major technical decisions
**Duration:** 60-90 minutes
**Attendees:** Technical team, architects, technical stakeholders

### Pre-Release Review
**Focus:** Release readiness, final validations
**Frequency:** Before each major release
**Duration:** 60 minutes
**Attendees:** Full team + release approval stakeholders

## Tips for Effective Stakeholder Reviews

### For Meeting Organizers
- Send materials with enough lead time for review
- Set clear objectives and desired outcomes
- Keep presentations focused and concise
- Use visuals and demos over long presentations
- Manage time strictly to respect attendees' schedules
- Follow up promptly with notes and action items

### For Presenters
- Practice demo beforehand
- Have backup plans for technical issues
- Focus on outcomes over technical details
- Be honest about challenges and risks
- Come prepared with options for decisions
- Listen actively to stakeholder feedback

### For Note Takers
- Capture key discussion points
- Document all decisions with context
- Record action items with clear owners and dates
- Note any dissenting opinions or concerns
- Distribute notes within 24 hours
- Use consistent format for easy reference

### For Stakeholders
- Review materials before the meeting
- Come prepared with questions and feedback
- Provide specific, actionable feedback
- Make decisions when possible
- Communicate concerns clearly
- Commit to action items when appropriate

## Common Pitfalls to Avoid

- Running over time without covering key topics
- Getting lost in technical details
- Presenting problems without proposed solutions
- Not documenting decisions clearly
- Failing to identify action item owners
- Not following up on previous action items
- Overloading the agenda
- Not leaving time for discussion and questions

## Meeting Success Criteria

- [ ] All agenda items covered within allocated time
- [ ] Key decisions documented with owners
